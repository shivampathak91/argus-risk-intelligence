# PROJECT GOAL

ARGUS is an AI-powered Early Warning & Risk Intelligence Platform.

It analyzes disaster and infrastructure incidents using multiple AI agents.

The platform should help governments and organizations detect risks early, simulate outcomes, prioritize actions, and generate explainable recommendations.

The application should feel like an AI Mission Control Center rather than a chatbot.

# REAL AI IMPLEMENTATION (MANDATORY)

This project must perform real analysis and reasoning.

It must NOT be a UI showcase or a fake demo.

Never use hardcoded outputs, random values, mock responses, placeholder logic, or simulated AI decisions unless explicitly in Demo Mode.

When a user uploads data, the system must genuinely process it and produce results derived from the uploaded content.

Examples

Image Upload
- Analyze the uploaded image using the Vision Agent.
- Detect visible objects, damage, or anomalies.
- Base the risk assessment on the detected features.
- Different images should produce different outputs.

PDF Upload
- Extract text using OCR.
- Parse the report.
- Identify important entities, measurements, dates, and findings.
- Use the extracted information in downstream agents.

CSV Upload
- Parse the dataset.
- Validate the data.
- Compute real statistics, trends, anomalies, and risk indicators.
- Use the data for simulations and recommendations.

TXT Upload
- Read and understand the content.
- Extract relevant information.
- Include it in agent reasoning.

Knowledge Agent
- Retrieve relevant historical records from SQLite.
- Compare with previous incidents.
- Use historical evidence when generating recommendations.

Simulation Agent
- Generate outcomes based on uploaded data and calculated risk.
- Simulations must change when inputs change.
- Never return fixed or pre-written results.

Recommendation Agent
- Recommendations must be supported by evidence.
- Explain why each recommendation was made.
- Include a confidence score.

Commander Agent
- Combine outputs from all agents.
- Resolve conflicting recommendations.
- Produce a final explainable decision.

# EXPLAINABLE AI

Every recommendation must include:

- Evidence used
- Contributing agents
- Confidence score
- Reasoning
- Data sources
- Historical matches (if any)

The user should always understand WHY the system produced its result.

# VALIDATION

The platform must produce different outputs for different uploads.

Example

Bridge Image A → Minor cracks → Medium Risk

Bridge Image B → Severe cracks → Critical Risk

Flood Report A → Low rainfall → Moderate Alert

Flood Report B → Extreme rainfall → High Alert

Results must be derived from the uploaded data, not from predefined templates.
